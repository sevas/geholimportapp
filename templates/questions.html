Where do you get your data from and why should I trust the schedules I get from this service?

We fetch the data directly from the GeHoL service. The HTML content is then parsed using the popular BeautifulSoup python library. Events are then created and converted to iCalendar or CSV files. As far as we're concerned, these files are just another way to present the exact same information.

If you won't take our word for it, you can have a look at the code of our underlying library on this bitbucket repository. You can even contribute, if you think it's missing something. The code for this entire web application should be released under the same terms, as soon as possible.


What do you mean by "complete GeHoL URL" ?

This service can get you two types of schedules. The first one is per-course schedule. Using the course mnemonic, we can easily reconstruct the complete URL and fetch the data for you. The second type of schedules, however, is problematic, and we need a little work on your part to generate the iCalendar files.

You'll need to go here and select the values of interest as you normally would. The first two values are mandatory. The third one should be selected only if it makes sense to you. The fourth value won't be used, as we will generate events for the whole term. At the end, you should get something like this:

Something went wrong, what can I do?

We tried to test this service with many different kind of schedules, from every faculty. However, parsing that kind of HTML is very fragile. Even more so when it sometimes comes back with incomplete data. We will soon add some sort of feedback strategy, so you can report unsuccessful queries.


The GeHoL server is down, now what?

We have no control over that server. Sometimes it gets disconnected from the internet with no indication of what is going on. We try at least to show it clearly in the sidebar on the right.
